# Production Setup Guide: GitHub Actions, Neon, and Render

This guide outlines the step-by-step process to deploy the Internal Tool Directory web application to production using GitHub Actions for Continuous Integration (CI), Neon for the PostgreSQL database, and Render for Continuous Deployment (CD).

## 1. Set Up Neon PostgreSQL Database

Neon provides a serverless PostgreSQL database for our production environment.

1. Go to [Neon.tech](https://neon.tech/) and sign up or log in.
2. Click **Create a Project**.
3. Name your project (e.g., `internal-tool-directory`), select a Postgres version, and choose a region close to your deployment (e.g., US East, Europe).
4. Once the project is created, navigate to the **Dashboard**.
5. Locate the **Connection Details** section.
6. Copy the **Postgres connection string** (it will look like `postgresql://[user]:[password]@[host]/[dbname]?sslmode=require`).
7. Save this connection string securely; you will need it as an environment variable in Render.

## 2. Set Up GitHub Actions (CI)

GitHub Actions acts as the testing gatekeeper, automatically running your Pytest suite on every push to the repository.

1. Ensure your CI workflow file is located at `.github/workflows/ci.yml` in your repository.
2. According to the PRD, the pipeline spins up a PostgreSQL service container, runs the synthetic data `seed.py` script, and executes automated pytest suites.
3. **Configure GitHub Secrets**:
   - Go to your GitHub repository in the browser.
   - Navigate to **Settings** > **Secrets and variables** > **Actions**.
   - Click **New repository secret**.
   - Add the necessary secrets defined in your `ci.yml` (e.g., `POSTGRES_PASSWORD`).
4. Push your codebase to the `main` branch. 
5. Go to the **Actions** tab in your GitHub repository and verify that the test suite passes successfully.

## 3. Set Up Render (CD)

Render will host the Flask application using the Dockerfile you have prepared.

1. Go to [Render.com](https://render.com/) and sign in (preferably with your GitHub account).
2. Click **New +** and select **Web Service**.
3. Connect your GitHub repository containing the project.
4. Configure the Web Service:
   - **Name**: `internal-tool-directory` (or your preferred name).
   - **Branch**: `main`.
   - **Environment**: Select **Docker** (since your Dockerfile is ready).
   - **Region**: Choose the same or closest region to your Neon database to minimize latency.
5. **Configure Environment Variables**:
   - Scroll down to the **Environment Variables** section and click **Add Environment Variable**.
   - Add the following keys:
     - **Key**: `DATABASE_URL` | **Value**: *[Paste the Neon connection string from Step 1]*
     - **Key**: `SECRET_KEY` | **Value**: *[Generate a strong random string for Flask sessions]*
6. **Automated State Management (The Destructive Build)**:
   - As per the TDD, Render needs to run `python seed.py` to drop/recreate tables and populate synthetic testing data.
   - **If using Render UI**: Since it's a Docker environment, make sure your Docker startup script (like an `entrypoint.sh` or CMD) runs `python seed.py` before starting the Flask server (e.g., `python seed.py && gunicorn app:app`).
   - **If using `render.yaml`**: Ensure your Infrastructure as Code file includes the pre-deploy command: `preDeployCommand: "python seed.py"`.
7. Click **Create Web Service**.
8. Render will now pull your code, build the Docker image, and deploy the application. 
9. Once the build finishes and the service starts, Render will provide a live URL (e.g., `https://internal-tool-directory.onrender.com`).

## 4. Verification & Testing

1. Navigate to the Render live URL provided in the Render dashboard.
2. Verify that the login screen loads correctly.
3. Log in with the default Admin credentials generated by your `seed.py` script.
4. Attempt to add, edit, and delete a tool to ensure the application is successfully communicating with the remote Neon PostgreSQL database.
5. Verify that standard users cannot access the `/admin` routes and see the explicit Security Alert view as required for your assignment video evidence.
